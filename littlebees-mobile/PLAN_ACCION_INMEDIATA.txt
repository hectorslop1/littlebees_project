PASO 2: PROBAR ENDPOINTS MANUALMENTE
------------------------------------
# Login
curl -X POST http://localhost:3001/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"padre@littlebees.com","password":"Parent123!"}'

# Copiar el accessToken de la respuesta

# Obtener hijos (usar token del paso anterior)
curl http://localhost:3001/children \
  -H "Authorization: Bearer <TU_TOKEN_AQUI>"

# Si responde con datos JSON → Backend funciona ✅
# Si responde con [] → Falta ejecutar seed
# Si responde 401 → Token inválido
# Si responde 404 → Endpoint no existe

PASO 5: VERIFICAR LOGS
-----------------------
En la consola de Flutter, buscar:

✅ ÉXITO:
[API] GET /children
[API] Response: [{id: "...", firstName: "..."}]

❌ ERROR:
[API] DioException: Connection refused
→ Backend no está corriendo

[API] Response: 401 Unauthorized
→ Token inválido o expirado

[API] Response: []
→ No hay datos, ejecutar seed

🔧 SOLUCIONES POR TIPO DE ERROR
================================

ERROR: "Connection refused" o "Failed to connect"
--------------------------------------------------
CAUSA: Backend no está corriendo
SOLUCIÓN:
1. cd littlebees-web/apps/api
2. npm run dev
3. Verificar que muestre puerto 3001

ERROR: 401 Unauthorized
-----------------------
CAUSA: Token inválido o no se está enviando
SOLUCIÓN:
1. Verificar que login funcione
2. Revisar que token se guarde en SecureStorage
3. Verificar AuthInterceptor en lib/core/api/auth_interceptor.dart

ERROR: Response vacía []
------------------------
CAUSA: No hay datos en base de datos
SOLUCIÓN:
1. cd littlebees-web/apps/api
2. npm run seed
3. Reiniciar app móvil

ERROR: Pantallas vacías pero sin errores en logs
-------------------------------------------------
CAUSA: UI no está renderizando datos correctamente
SOLUCIÓN:
1. Verificar que providers estén retornando datos
2. Agregar prints en repositorios:
   print('Children loaded: ${children.length}');
3. Verificar widgets de error/empty state
